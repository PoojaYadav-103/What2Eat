ChatGPT said:

Great question! Here's a clear explanation:

📝 Why Use git commit -m "Add website files"?

✅ Purpose of git commit:

A commit in Git is like taking a snapshot of your project at a moment in time.
You're telling Git:

“I’m saving this version of all the changes I just added.”

🧠 Breaking Down the Command: 

bash
git commit -m "Add website files"

  • git commit → saves the current changes

  • -m → means "message"

  • "Add website files" → a message describing what this commit does

📌 Why the Message Matters:

  • It helps you and others understand what changed and why.

  • Acts like a label for the snapshot.

  • Good habit for team projects and version tracking.

📦 Without committing:

If you skip this, your files are not saved in Git history, and you can't push them to GitHub.

In short:

💬 Commits = Saving points + Clear history of changes
